# 테스트 환경 구축 가이드

> 맵과 오브젝트를 넣어서 바로 카메라 테스트를 할 수 있는 환경을 만드는 방법입니다.
> 언리얼 담당자 혼자서도 진행할 수 있어요!

---

## 목표

이 가이드를 완료하면 이런 테스트 환경이 완성됩니다:

```
[완성 상태]
  ✅ 걸어다닐 수 있는 맵 (야외 or 실내)
  ✅ 움직이는 캐릭터 (플레이어 + NPC)
  ✅ AI 카메라가 자동으로 앵글을 바꿈
  ✅ 대화 / 이동 씬 전환 테스트 가능
```

### B-3. 기본 환경 세팅 (바닥 + 조명)

#### 바닥 만들기

1. Place Actors 패널 열기 (`Shift + 1`)
2. **Plane** 검색 → 뷰포트에 드래그
3. Details 패널에서 크기 조절:
   - Scale: X=50, Y=50, Z=1 (넓은 바닥)
4. 재질(Material) 적용:
   - Content Browser → `StarterContent/Materials/M_Ground_Grass` 찾기
   - 바닥 메시에 드래그 앤 드롭

#### 조명 세팅

```
필수 조명 요소 3가지 (없으면 화면이 검거나 너무 밝음):

1. Directional Light (태양광)
   - Place Actors → "Directional Light" 검색 → 배치
   - Details: Intensity = 10, Color = 따뜻한 노란빛

2. Sky Atmosphere (하늘)
   - Place Actors → "Sky Atmosphere" → 배치

3. Sky Light (환경광)
   - Place Actors → "Sky Light" → 배치
   - Details: Real Time Capture = 체크
```

#### 간단한 장애물/구조물 추가

카메라 테스트를 다양하게 하려면 오브젝트가 있어야 해요:

```
Place Actors에서 Shapes 카테고리:
  - Cube (큐브): 건물/벽 대용
  - Cylinder (원기둥): 기둥
  - Sphere (구): 환경 오브젝트

Starter Content에서:
  Content/StarterContent/Props/ 폴더의 소품들 활용
```

**배치 팁:**
- 막힌 공간(건물 안): 대화 씬 테스트용
- 탁 트인 공간(광장): 전투/탐험 씬 테스트용
- 높낮이 차이: 로우앵글/버즈아이 앵글 테스트용

### B-4. 캐릭터 배치

#### 플레이어 캐릭터

Third Person 기본 캐릭터를 그대로 써도 되고,
Paragon Wraith를 캐릭터로 쓰고 싶다면:

1. `Content/ParagonWraith/Characters/Heroes/Wraith/` 열기
2. `BP_Wraith` (블루프린트) 찾기
3. 뷰포트에 드래그

**플레이어 캐릭터로 지정하기:**
1. 해당 블루프린트 더블클릭
2. Details → `Pawn → Auto Possess Player → Player 0` 설정
3. 기존 `BP_ThirdPersonCharacter`의 Auto Possess는 `Disabled`로 변경

#### NPC 캐릭터 (대화 씬 테스트용)

1. Content Browser에서 캐릭터 블루프린트 찾기
2. 레벨 뷰포트에 드래그 (플레이어와 마주보게 배치)
3. 이 캐릭터는 `Auto Possess → Disabled` (플레이어가 조종 안 함)
4. 두 캐릭터를 2~3미터 거리로 마주보게 배치

### B-6. 테스트 씬 구분 구역 만들기

맵에 구역을 나눠서 만들면 씬 타입별 테스트가 쉬워요:

```
[맵 레이아웃 예시]

  ┌──────────────┬──────────────┐
  │              │              │
  │  대화 구역   │  전투 구역   │
  │  (좁은 방)   │  (넓은 공터) │
  │              │              │
  ├──────────────┴──────────────┤
  │                             │
  │       이동/탐험 구역         │
  │      (길게 뻗은 복도)        │
  │                             │
  └─────────────────────────────┘
```

**구역 만드는 법:**
- 큐브 메시로 벽 만들기 (Scale 조절)
- 각 구역 사이에 통로 만들기

## 방법 C — 이미 완성된 맵 사용 (가장 품질 높음)

일부 에셋은 이미 완성된 데모 맵(`.umap`)을 포함해요.
이 경우 맵을 직접 열어서 AI 카메라만 추가하면 됩니다.

### C-1. 완성된 맵이 있는 에셋들

```
- Soul: Cave → Maps/ 폴더에 완성 맵 포함
- City Sample (대규모, 무거움) → 고품질 도시 맵
- Electric Dreams Environment → 판타지 환경 (무료)
- Lyra Starter Game → 전투 테스트용 (무료)
```

### C-2. 완성 맵에 AI 카메라 추가하기

1. 해당 `.umap` 파일 더블클릭으로 열기
2. `CineAI Camera` 액터 배치 (씬 중앙 부근)
3. `Cine AI Director` 액터 배치
4. Director의 Target Camera 연결
5. Play 테스트!

## 테스트 체크리스트

테스트 환경이 완성됐는지 확인하세요:

```
환경 체크:
  [ ] 바닥이 있고 캐릭터가 걸어다닐 수 있다
  [ ] 조명이 있어서 화면이 밝게 보인다
  [ ] 다양한 구조물/오브젝트가 배치되어 있다

캐릭터 체크:
  [ ] WASD로 조종되는 플레이어 캐릭터가 있다
  [ ] NPC 캐릭터가 1명 이상 있다 (대화 씬용)

카메라 시스템 체크:
  [ ] CineAI Camera 액터가 레벨에 있다
  [ ] Cine AI Director 액터가 있다
  [ ] Director의 Target Camera가 연결됐다
  [ ] Play 시 출력창에 에러가 없다

테스트 가능 씬:
  [ ] 캐릭터가 이동할 때 카메라 반응 확인
  [ ] NPC 옆에 서면 대화 씬 인식 확인 (ML 담당자와 함께)
```

## 자주 하는 실수

**맵이 너무 어두워요**
→ Sky Light의 `Real Time Capture`를 체크했는지 확인
→ `Build → Build Lighting Only` 실행해보기

**캐릭터가 바닥 아래로 떨어져요**
→ Player Start 위치가 바닥 위에 있는지 확인
→ 바닥 메시에 Collision이 켜져 있는지 확인
   (메시 선택 → Details → Collision → `Block All`)

**에셋이 Content Browser에 없어요**
→ Launcher에서 해당 프로젝트로 Add가 완료됐는지 확인
→ 에디터에서 `Edit → Refresh Asset Registry`

**플레이하면 카메라가 기본 ThirdPerson 카메라로 보여요**
→ BP_ThirdPersonCharacter 블루프린트를 열어서
   `CameraComponent` 또는 `SpringArm`의 설정 확인
→ 우리 `CineAICameraActor`를 현재 뷰 카메라로 활성화해야 해요
   → `Window → Level Viewport → Camera → CineAI Camera`
