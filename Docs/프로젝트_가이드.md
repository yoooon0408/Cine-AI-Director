# Cine AI Director — 프로젝트 가이드

> **이 문서 하나로 전체 프로젝트를 파악할 수 있습니다.**
> UE5 담당자가 처음 읽어도 이해할 수 있도록 작성했습니다.

---

## 이 프로젝트가 하는 일

**영화 제작 환경**에서 AI가 실시간으로 카메라 샷을 자동 선택해주는 UE5 플러그인입니다.

```
씬 상황 분석 → AI가 샷 타입 결정 → 카메라 자동 이동
   (대화인가?      (클로즈업이 적절)    (부드럽게 전환)
    액션인가?
    탐험인가?)
```

씬 타입별 선택 가능한 샷:

| 씬 타입 | 추천 샷 |
|---------|---------|
| 대화(Dialogue) | CloseUp, MediumCloseUp, OverShoulder, TwoShot |
| 액션(Combat) | Wide, Medium |
| 탐험(Exploration) | Wide, ExtremeWide, BirdsEye |
| 사망(Death) | CloseUp, LowAngle |
| 승리(Victory) | Wide, BirdsEye |

## UE5 담당자가 해야 할 일 (단계별)

### Step 1 — 플러그인 설치

1. 이 플러그인 폴더 전체를 UE5 프로젝트의 `Plugins/` 폴더에 복사
2. UE5 편집기를 열면 자동으로 플러그인이 인식됩니다
3. Edit > Plugins > "Cine AI Director" 활성화 후 재시작

---

### Step 2 — 레벨 세팅 (필수 2개 액터)

**레벨에 반드시 2개 액터를 배치해야 합니다:**

#### ① CineAICameraActor
- Place Actors 패널에서 "Cine AI Camera" 검색 후 레벨에 드래그
- 초기 위치는 아무 데나 둬도 됩니다 — AI가 자동으로 이동시킵니다

#### ② AIDirector
- Place Actors 패널에서 "Cine AI Director" 검색 후 레벨에 드래그
- Details 패널에서 설정:

| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| **Target Camera** | 위에서 배치한 CineAICameraActor를 드래그해서 연결 | **필수** |
| **API Key** | Claude AI API 키 입력 | 선택 (없으면 규칙 기반으로 작동) |
| **Decision Interval** | AI가 샷을 결정하는 주기 (초, 기본 3초) | 선택 |
| **Min Shot Duration** | 최소 샷 유지 시간 (기본 2초) | 선택 |
| **bUseLocalInferenceServer** | 로컬 ML 모델 사용 여부 (ML 담당자와 협의) | 선택 |

---

### Step 3 — 씬 타입 설정 (중요)

AI가 어떤 상황인지 알아야 적절한 샷을 선택할 수 있습니다.

**방법 A — 블루프린트로 직접 설정 (권장)**

씬 전환 이벤트(예: 대화 시작, 액션 시작)가 발생하는 블루프린트에서:
```
SceneAnalyzer → SetSceneTypeOverride(Dialogue)  ← 대화 시작 시
SceneAnalyzer → ClearSceneTypeOverride()          ← 대화 종료 시
```

**방법 B — Details 패널에서 고정**

AIDirector 안의 SceneAnalyzer 컴포넌트 선택 → Details 패널:
- `bOverrideSceneType` 체크
- `OverrideSceneType` 에서 원하는 씬 타입 선택

**방법 C — 자동 감지 (인터페이스 미구현 시)**

- 캐릭터들이 빠르게 이동 중 → Exploration으로 자동 분류
- 캐릭터들이 정지 중 + 2명 이상 → Dialogue로 자동 분류

---

### Step 4 — 캐릭터 인터페이스 추가 (선택)

더 정확한 씬 분류가 필요할 때 사용합니다.

캐릭터 블루프린트에서:
1. **Class Settings** → **Implemented Interfaces** → `CineCharacterInterface` 추가
2. **My Blueprint** → **Interfaces** 탭에 4개 함수가 생성됩니다:

| 함수 | 반환값 | 구현 예시 |
|------|--------|----------|
| `GetHealthNormalized` | 0.0 ~ 1.0 | 영화에서는 사용 안 해도 됨 |
| `IsSpeaking` | bool | 대화 시스템 활성화 여부 |
| `IsInCombat` | bool | 액션 시퀀스 여부 |
| `IsDead` | bool | 영화에서는 사용 안 해도 됨 |

> **영화 환경 참고:** 게임과 달리 실제 체력/사망은 의미 없습니다.
> `IsSpeaking`과 `IsInCombat`만 구현해도 충분합니다.

---

### Step 5 — 카메라를 플레이어 뷰로 설정

UE5가 게임 실행 시 이 카메라를 사용하도록 설정해야 합니다.

**방법 A — GameMode 또는 PlayerController에서:**
```
PlayerController → SetViewTargetWithBlend(CineAICameraActor, BlendTime)
```

**방법 B — 레벨 블루프린트에서:**
```
BeginPlay → GetPlayerController → SetViewTarget(CineAICameraActor)
```

## 각 C++ 파일 상세 설명

### `ShotTypes.h` — 열거형 정의
- `ECineShotType`: 13가지 샷 타입 (None ~ Dutch)
- `ECineSceneType`: 7가지 씬 타입 (Unknown ~ Victory)
- `ECineTransitionType`: 3가지 전환 방식 (Cut / Blend / Fade)
- **⚠ 수정 시 ML 담당자에게 알려야 합니다 — Python `reward_model.py`와 동기화 필요**

### `SceneContext.h` — 씬 상태 데이터
- `FSceneContext`: AI에게 전달되는 씬 전체 스냅샷
- `FCineCharacterInfo`: 캐릭터별 위치/속도/상태 정보
- **UE5 ↔ ML 경계 구조체** — 필드 추가 시 ML 담당자와 협의 필요

### `DirectorDecision.h` — AI 결정 데이터
- `FDirectorDecision`: AI가 내린 카메라 결정 (샷 타입 + 목표 위치 + 전환 방식)
- `IsValid()`: ShotType이 None이 아니고 Confidence > 0 이면 유효

### `CineAICameraActor.h/.cpp` — 카메라 액터
- `ACineCameraActor` 상속 (UE5 내장 시네마틱 카메라)
- `ApplyDecision()`: AIDirector가 호출 → 목표 위치로 부드럽게 이동
- `IsTransitioning()`: 이동 중인지 블루프린트에서 확인 가능
- Cut / Blend 전환 구현 완료 | **Fade는 미구현 (차후 작업 참고)**

### `ShotBlender.h/.cpp` — 블렌딩 계산
- 트랜지션 타입별 알파값 계산 (EaseInOut, Linear)
- `CineAICameraActor`가 내부적으로 사용 (직접 호출 불필요)

### `ICineCharacterInterface.h` — 캐릭터 블루프린트 인터페이스
- 구현하면 씬 분류 정확도 향상
- 구현 안 해도 이동 속도 기반 자동 분류로 정상 작동

### `SceneAnalyzer.h/.cpp` — 씬 분석 컴포넌트
- `CaptureContext()`: 매 분석 주기마다 씬 상태 스냅샷 생성
- `ClassifyScene()`: 사망 > 전투 > 대화 > 탐험 우선순위로 씬 분류
- `SetSceneTypeOverride()` / `ClearSceneTypeOverride()`: 블루프린트에서 씬 타입 직접 지정

### `AIDirector.h/.cpp` — 핵심 AI 디렉터 액터
- 매 `DecisionInterval`마다 씬 분석 → AI API 호출 → 카메라 이동
- Claude API 모드 / 로컬 추론 서버 모드 / 규칙 기반 폴백 모드 3가지 지원
- `ComputeShotTransform()`: 샷 타입 → 실제 UE5 카메라 위치 계산 (12종)
- `OnDecisionMade`: 결정이 내려질 때 블루프린트 이벤트 발생

### `ExperienceLogger.h/.cpp` — 학습 데이터 수집
- 매 AI 결정마다 `(씬 상태, 샷 결정, 리워드)` 를 JSONL로 저장
- 저장 위치: `프로젝트/Saved/CineAILogs/session_<ID>.jsonl`
- ML 학습 데이터의 원천

### `RewardEvaluator.h/.cpp` — 품질 점수 계산
- `EvaluateDecision()`: 카메라 결정의 품질을 -1.0 ~ 1.0으로 평가
- 프레이밍 점수 + 샷 적합성 점수 + 샷 유지 시간 점수 합산

## Python ML 파이프라인 설명 (ML 담당자용)

### 학습 흐름

```
UE5 게임 실행
    ↓
Saved/CineAILogs/*.jsonl 생성
    ↓
python run_pipeline.py --mode dataset
    ↓ (collector.py → preprocessor.py)
data/processed/*.npy 생성
    ↓
PPO 학습 (trainer.py)
    ↓
models/exports/cine_director_final.zip
    ↓
python -m inference.server 실행
    ↓
UE5 bUseLocalInferenceServer = true 설정
```

### 즉시 학습 (UE5 데이터 없이도 가능)

```bash
cd ML
pip install -r requirements.txt
python src/run_pipeline.py --mode synthetic --timesteps 200000
```

### UE5 로그 데이터로 학습

```bash
# 1. UE5에서 게임 실행 → Saved/CineAILogs/ 에 로그 쌓임
# 2. 로그를 ML/data/raw/ 에 복사
# 3. 학습 실행
python src/run_pipeline.py --mode dataset --timesteps 500000
```

### 로컬 추론 서버 실행

```bash
python -m src.inference.server
# → http://127.0.0.1:8765/predict 로 서비스
# → UE5 AIDirector > bUseLocalInferenceServer = true 설정
```

## 차후 작업해야 할 내용

### 🔴 높은 우선순위

| 항목 | 담당 | 설명 |
|------|------|------|
| **Fade 전환 구현** | UE5 | `CineAICameraActor.cpp`의 Fade 케이스가 현재 Blend와 동일하게 동작함. 실제 페이드 아웃/인은 Post Process Volume의 씬 노출 또는 UMG 오버레이를 이용해야 함 |
| **PlayerController 연동** | UE5 | 플레이어 카메라를 CineAICameraActor로 전환하는 블루프린트 작성 필요 |
| **RewardEvaluator 연동** | ML | `AIDirector.cpp`가 결정 후 `RewardEvaluator::EvaluateDecision()`을 호출해서 즉시 리워드를 로깅하도록 연결 (현재 reward=0으로 기록됨 — Python이 재계산하므로 학습에는 문제없음) |

### 🟡 중간 우선순위

| 항목 | 담당 | 설명 |
|------|------|------|
| **씬 타입 블루프린트 이벤트 자동화** | UE5 | Sequencer 트랙이나 레벨 블루프린트 타임라인에서 자동으로 SetSceneTypeOverride 호출 |
| **카메라 POV 구현** | UE5 | POV 샷 시 캐릭터 본(head bone)에 카메라를 어태치해야 정확함. 현재는 캐릭터 눈 높이 위치로 계산 |
| **Claude API 키 보안** | ML | 현재 Details 패널에 평문 저장. UE5의 `FPlatformMisc::GetEnvironmentVariable()`로 환경변수에서 로드하도록 변경 권장 |
| **모델 정규화 통계 저장** | ML | `preprocessor.py`가 매번 데이터 분포에서 mean/std를 계산함. 학습 시의 통계를 파일로 저장해두고 추론 서버에서 동일한 값으로 정규화해야 함 (현재는 synthetic 스케일 고정값 사용) |
| **다중 카메라 지원** | 양쪽 | 현재 카메라 1대만 제어 가능. 여러 카메라 앵글을 등록하고 AI가 전환하는 구조로 확장 |

### 🟢 낮은 우선순위

| 항목 | 담당 | 설명 |
|------|------|------|
| **Editor 위젯** | UE5 | 에디터에서 샷 타입 프리뷰, 리워드 실시간 확인 창 추가 |
| **RLHF (인간 피드백)** | ML | 현재 규칙 기반 리워드를 사람이 직접 평가한 점수로 교체하는 파이프라인 |
| **ONNX 모델 내보내기** | ML | Python 서버 없이 UE5 내에서 직접 추론 가능하도록 ONNX 변환 후 `NNE` 플러그인 연동 |

## UE5 ↔ ML 동기화 주의사항

이 두 파일 쌍은 **항상 같은 값**을 유지해야 합니다:

| UE5 파일 | Python 파일 | 동기화 항목 |
|----------|------------|------------|
| `Data/ShotTypes.h` | `reward_model.py` | ShotType / SceneType enum 숫자 |
| `Data/SceneContext.h` | `preprocessor.py` STATE_COLUMNS | 씬 상태 필드 이름 및 순서 |
| `Logging/ExperienceLogger.cpp` | `collector.py` | JSONL 필드 이름 구조 |
| `Logging/RewardEvaluator.cpp` | `reward_model.py` | 리워드 가중치 (현재 framing=0.4, appropriateness=0.4, duration=0.2) |

**변경 시 반드시 상대방에게 알리고 두 파일 동시에 수정하세요.**
