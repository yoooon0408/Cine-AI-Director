**Cine AI Director**는 게임 영상을 자동으로 멋지게 찍어주는 AI 카메라 시스템이에요.

영화 감독이 "지금 이 장면은 클로즈업으로 찍어야지" 라고 판단하는 것처럼,
AI가 게임 속 상황을 보고 스스로 최적의 카메라 앵글을 골라줍니다.

```
[게임 플레이 중]
  캐릭터가 대화 중 → AI가 판단 → 오버숄더 샷으로 전환
  전투가 시작됨   → AI가 판단 → 와이드 샷으로 전환
  보스가 등장함   → AI가 판단 → 로우앵글 드라마틱 샷
```

## 역할

```
┌─────────────────────────────────┬──────────────────────────────────┐
│         언리얼 담당              │           ML 담당                │
│   (게임 세계를 만드는 사람)       │   (AI 두뇌를 만드는 사람)        │
├─────────────────────────────────┼──────────────────────────────────┤
│ • 레벨(맵) 꾸미기                │ • AI API 연동                    │
│ • 캐릭터, 오브젝트 배치          │ • 데이터 수집/분석               │
│ • 카메라 액터 배치               │ • 강화학습 모델 훈련              │
│ • 씬 분류 로직 조정              │ • 리워드 함수 설계               │
│ • 에디터 UI                     │ • Python 파이프라인              │
└─────────────────────────────────┴──────────────────────────────────┘
                          ↕ 데이터로 연결
                   [ExperienceLogger]
              (언리얼이 찍은 데이터를 ML에게 전달)
```

## 개발 순서 (큰 그림)

```
Phase 1 │ 언리얼 담당: 테스트 맵 구성 + 캐릭터 배치
        │ ML 담당:     Claude API 연동 + 기본 샷 선택 구현
        │
Phase 2 │ 언리얼 담당: 씬 분류 로직 다듬기 (전투/대화 감지)
        │ ML 담당:     데이터 로깅 확인 + 리워드 설계
        │
Phase 3 │ 언리얼 담당: 카메라 연출 퀄리티 향상
        │ ML 담당:     수집된 데이터로 자체 RL 모델 훈련
        │
Phase 4 │ 함께:        통합 테스트 + 포트폴리오 영상 촬영
```

## Git으로 협업하는 법

Git이 처음이라면 이것만 기억하세요.

### 기본 흐름

```
작업 전:  git pull              ← 상대방 작업 내용 받아오기
작업 중:  파일 수정
작업 후:  git add .
          git commit -m "설명"   ← 내 작업 저장
          git push               ← 상대방에게 보내기
```

### 브랜치 사용법

```bash
# 언리얼 담당이 새 작업 시작할 때
git checkout -b feature/ue5-test-map

# ML 담당이 새 작업 시작할 때
git checkout -b feature/ml-api-integration

# 작업 완료 후 → GitHub에서 Pull Request 만들기
# → 상대방이 확인 후 main에 합치기
```

### 충돌(Conflict)이 났을 때

같은 파일을 두 사람이 동시에 수정하면 충돌이 납니다.
당황하지 말고 상대방에게 알리고 같이 해결하세요.
**혼자 해결하려다 파일 날리는 경우가 많아요!**

## 소통 규칙

| 상황 | 해야 할 일 |
|------|-----------|
| `ShotTypes.h` 파일에 항목 추가 | ML 담당에게 바로 알리기 |
| `SceneContext.h` 필드 추가/삭제 | ML 담당에게 바로 알리기 |
| API 연동 방식 변경 | 언리얼 담당에게 바로 알리기 |
| 로그 포맷 변경 | 상대방에게 바로 알리기 |

> 작은 변경도 공유해주세요. 연결 지점이 어긋나면 디버깅이 매우 힘들어요.
